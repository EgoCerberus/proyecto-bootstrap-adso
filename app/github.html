Guía de Configuración Local y Remota

Sigue estos pasos secuenciales ejecutando únicamente los siguientes comandos en tu consola.

Paso 1: Verificar la instalación de Git

Comprueba que Git está instalado correctamente en tu equipo ejecutando el comando correspondiente en la terminal o CMD.

git --version

Paso 2: Configurar tu identidad

Establece tu nombre de usuario y correo electrónico global para que Git pueda identificar quién realiza los commits en el historial.

git config --global user.name "TuNombre" git config --global user.email "tu@email.com"

Paso 3: Iniciar un proyecto local

Crea o abre una carpeta de proyecto y conviértela en un repositorio Git local mediante la consola para comenzar el control de versiones.

cd mi-proyecto git init

Paso 4: Agregar archivos y realizar el primer commit

Añade los archivos del proyecto al área de preparación (Staging Area) y guarda una primera versión estable con un mensaje descriptivo.

git add . git commit -m "Mi primer commit"

Paso 5: Crear un repositorio en GitHub

Accede a tu cuenta de GitHub desde el navegador y crea un nuevo repositorio vacío donde se almacenará el proyecto de forma remota.

Accede a: github.com/new

Asigna el nombre a tu repositorio y presiona "Create repository".

Paso 6: Conectar el proyecto local con GitHub

Vincula el repositorio local con la URL del repositorio remoto en GitHub y envía tus ramas principales para sincronizar ambos entornos.

git remote add origin https://github.com/TuUsuario/repo.git git branch -M main git push -u origin main

Paso 7: Proyecto publicado

El repositorio queda correctamente conectado con GitHub, seguro en la nube y listo para seguir guardando cambios o colaborar en equipo.
